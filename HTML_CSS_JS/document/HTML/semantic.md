## What is Semantic HTML?

**Semantic HTML** means using HTML elements that clearly describe the **meaning and purpose** of the content they contain.

Instead of using generic tags like `<div>` and `<span>` for everything, semantic elements tell the browser and developers **what the content represents**.

In simple words:

> Semantic HTML = Meaningful HTML.

---

## Examples of Semantic Elements

| Tag         | Meaning                             |
| ----------- | ----------------------------------- |
| `<header>`  | Header section of a page or section |
| `<nav>`     | Navigation links                    |
| `<main>`    | Main content                        |
| `<section>` | Section of content                  |
| `<article>` | Independent content (blog, news)    |
| `<footer>`  | Footer                              |
| `<aside>`   | Sidebar                             |
| `<button>`  | Clickable button                    |
| `<form>`    | User input form                     |

```html
  </section>

  Copyright
</footer>
```

## Benefits of Semantic HTML

1. **Better Accessibility**
   Screen readers understand structure easily.

2. **Improved SEO**
   Search engines understand page hierarchy.

3. **Cleaner Code**
   Easier to read and maintain.

4. **Professional Standard**
   Used in real-world projects.

5. **Less CSS/JS Complexity**
   Meaning already defined by tags.

---

## Interview One-Line Answer

> Semantic HTML is the use of HTML elements that convey the meaning and structure of content clearly to browsers, developers, and assistive technologies.
